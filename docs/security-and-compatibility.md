# 安全与兼容性

本项目的配置策略以 Xray-core 正式版可解析、主流客户端可导入、运行时不生成明显错误配置为目标。它不能保证在任何网络环境中永久可用，也不能替代用户对本地法律法规、服务器网络和客户端兼容性的判断。

“增强能力”不是全部默认打开。脚本会优先保证域名、SNI、Host、证书、回落规则、分享链接和 Xray 配置保持一致；只有当前内核支持且配置测试通过时，才会启用更高级的能力。

## Xray-core 正式版能力

脚本会检测当前 Xray-core 是否支持相关命令和字段。支持时启用，不支持时提示升级，不生成当前内核无法运行的配置。

- VLESS Encryption：用于无回落的 VLESS WS/XHTTP。XHTTP 启用后会同步使用 Vision flow；如果配置测试不通过，会回退到兼容模式。
- REALITY ML-DSA-65：目标站点检查合适时启用，分享链接输出 `pqv`。目标证书链总长度需要达到 3500 以上，才会写入这个后量子签名校验参数。
- TLS ECH：支持时生成服务端 ECH 配置，分享链接输出 `ech`；配置测试失败时会自动清空 ECH，避免服务端和分享链接不一致。
- Hysteria2 QUIC 参数：当前内核支持时启用，不支持时跳过。默认只写 Xray-core 正式版支持的 Brutal 和端口跳跃参数。

建议通过菜单 `14.Xray-core 管理` 使用正式版。预览版入口只用于测试新能力；普通升级不会自动切到预览版。

## 为什么不是所有入口都启用 VLESS Encryption

部分 VLESS 入口需要承担浏览器回落或分流，不能同时开启 VLESS Encryption。脚本会按协议类型自动选择，避免生成 Xray-core 无法启动的配置。

XHTTP 是无回落入口，适合使用 VLESS Encryption。当前内核支持时，脚本会让 XHTTP 的服务端用户、分享链接和 Mihomo 订阅都带上 `flow: xtls-rprx-vision`；如果 Xray 配置测试失败，脚本会回退并保持分享输出与实际配置一致。

## REALITY 目标站点

Reality 目标站点应选择真实、稳定、TLS 行为合适的站点。脚本会做必要检查；不合适时会跳过对应高级项，并提示更换目标站点。

如果目标站点证书链长度低于 3500，脚本不会启用 ML-DSA-65。检测到 X25519MLKEM768 且证书链长度足够时，状态页会显示完整后量子增强条件满足；只满足证书链条件时，会显示为部分增强。这不等同于基础 Reality 是否可用，基础 Reality 仍以 TLS 1.3、证书域名和握手行为检查为准。

如果已在网站/反代管理中注册本机真实网站，脚本会优先把该站点域名作为 Reality 默认建议，但不会静默覆盖已有 Reality 配置。用户可以接受建议，也可以手动输入其他目标。

Reality `privateKey` 属于服务端敏感材料，不应出现在公开日志、截图或工单中。新版安装输出只展示脱敏值；如果历史安装日志曾经公开泄露完整 `privateKey`，应重新生成 Reality key/shortId，并同步更新所有客户端。

## TLS ECH

ECH 可以减少部分 TLS 明文信息暴露，但它不是万能保护。客户端需要支持分享链接中的 `ech` 字段；如果客户端不支持，可继续使用不带 ECH 的 TLS 分享链接。

脚本只会在 Xray-core 接受配置时保留 ECH。若 `xray -test` 不通过，安装或重配会回退到普通 TLS，并且分享链接不再输出 `ech`。

## Hysteria2

Hysteria2 默认使用 UDP/443，与 TCP/443 不冲突。启用端口跳跃后，Xray 仍只监听 UDP/443；脚本会把本机跳跃端口 REDIRECT 到 UDP/443，并把多端口范围写入分享链接。云安全组仍需要用户自行放行所选 UDP 端口范围。它适合 UDP 质量较好的线路，但对云防火墙、运营商 UDP 策略和多公网出口路由更敏感。带宽参数只写在服务端，不写进分享链接。

Hysteria2 的默认回落 URL 会优先使用已注册真实网站；没有真实网站时再使用历史配置、已有回落站点、Reality 目标或外部兜底。证书域名仍使用安装域名，回落 URL 可在安装或重配时手动覆盖。

Finalmask 的自定义握手、噪声、分片等高级外观参数不会默认启用。默认配置只使用正式版明确支持的参数，避免生成客户端或服务端不稳定的配置。

## 443 入口 PROXY protocol

PROXY protocol 是 443 入口的全局兼容模式，不能对不同 HTTPS 后端分别开关。脚本默认使用 `auto`：能安全开启时开启，检测到未知或不兼容的 HTTPS 后端时关闭。

菜单切换该模式时，会提示哪些后端会受到影响，并同步更新脚本管理的 Nginx 和 Xray 配置。宝塔、1Panel、OpenResty、Caddy、Apache 等第三方配置只做检测和接入提示，脚本不会自动改写它们的配置文件。

## VMess 兼容性

VMess WS TLS 保留是为了兼容旧客户端。新部署优先使用 VLESS、Reality、XHTTP 或 Hysteria2。

## 使用建议

- 使用真实域名和有效证书。
- 避免在同一台服务器上混用多个脚本管理同一套 Nginx/Xray 配置。
- 定期更新 Xray-core 正式版。
- 复杂网络环境中先确认 IPv4/IPv6、WARP、云防火墙和本机防火墙状态。
