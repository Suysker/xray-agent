# 证书管理

证书由菜单 `5.证书管理` 统一处理，默认存放在：

```text
/etc/xray-agent/tls
```

脚本会读取本地证书文件和 acme 记录，展示域名、剩余天数、证书与私钥是否匹配，以及是否存在续签记录。

## 申请方式

证书向导会先检查域名、解析、网络栈、TCP/80 和 TCP/443 端口，再推荐合适方式。

- HTTP-01 standalone：域名 A/AAAA 解析正确，TCP/80 可用时优先推荐。
- DNS-01：适合通配证书、无公网入口、解析不匹配、TCP/80 被占用或云防火墙限制的场景。
- 手动 TXT：适合不想保存 DNS API 密钥，或服务商暂不支持自动 API 的场景。

当前保留的 DNS-01 方式包括 Cloudflare、DNSPod、Aliyun、自定义 DNS 类型和手动 TXT。

## Hysteria2 证书

Hysteria2 需要用户自己控制的真实域名和 TLS 证书。安装或重配时，脚本会优先列出当前安装域名、当前 TLS 证书域名、本地证书和 acme 记录；你可以直接选择已有证书，也可以输入新的自定义域名。Reality 目标域名通常不是用户控制的域名，不能作为 Hysteria2 证书域名。

## TLS ECH

如果当前 Xray-core 和客户端支持 TLS ECH，脚本会在分享链接中输出对应参数。当本地证书可读时，分享链接也会附带证书校验信息，方便客户端校验证书。

## 续签

续签默认只处理已到期或 14 天内临期的证书。强制续签全部证书会增加触发 CA 限流的概率，执行前请确认确有需要。

## 常见失败原因

- 域名没有解析到当前服务器。
- 云防火墙没有放行 TCP/80 或 TCP/443。
- 本机已有服务占用验证端口。
- DNS TXT 记录未生效。
- DNS API 密钥权限不足或区域不匹配。
- ACME 邮箱格式错误或 CA 限流。

遇到失败时，先查看菜单提示和 `/etc/xray-agent/tls/acme.log`。
